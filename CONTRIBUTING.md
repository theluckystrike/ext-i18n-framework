# Contributing to ext-i18n-framework

Thank you for your interest in contributing.

## Reporting Issues

When reporting issues, please include:

- A clear description of the problem
- Steps to reproduce the issue
- Expected behavior vs actual behavior
- Environment details (Node version, OS, etc.)
- Any relevant error messages or logs

Use the issue tracker to report bugs and request features.

## Development Workflow

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch from main
4. Make your changes
5. Test your changes
6. Commit with a clear commit message
7. Push to your fork
8. Submit a pull request

## Code Style

- Use TypeScript for all new code
- Follow existing code formatting conventions
- Add type annotations where applicable
- Keep functions focused and modular
- Write clear, descriptive variable and function names

## Testing

Run tests before submitting changes:

```bash
npm test
```

Ensure all tests pass and no new warnings are introduced.
